# Havenor Build Phases

Havenor should grow in protected, observable phases.

This repository is public-facing. It should present the front-of-house experience while keeping protected organism internals outside the public repo.

---

## Phase 0 — Foundation / Identity

Status: started

Purpose:
- Establish Havenor as a conversation-first continuity ecosystem.
- Define the public/private boundary.
- Keep the repository clean, calm, and safe from the beginning.

Includes:
- README
- public positioning
- boundary language
- no private internals
- no credentials
- no Rock-core context

Exit condition:
- Anyone opening the repo can understand what Havenor is and what must not be exposed here.

---

## Permanent Rules

Never put these in this public repo:
- private memory
- protected continuity stores
- credentials or tokens
- internal prompts
- internal orchestration logic
- private operational doctrine
- Rock-core context
- sensitive system topology

The browser is the window.
It is never the vault.
